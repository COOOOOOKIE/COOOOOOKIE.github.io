Lists

今天重新回顾了一下链表(Lists)，有几点想法：

1.单链表的头指针和头结点


头指针是指向链表第一个结点的指针，头结点是链表的第一个结点。

头结点区别于链表的首元结点，链表的首元结点为第一个存有数据的节点。头结点的出现是为了使对第一个结点前插入和删除第一个结点的操作与其他结点统一。头结点的数据域一般无意义或放置链表长度等数据。头结点不是必须的。

头指针是指向链表第一个节点的指针，当有头结点时，头指针指向头结点。头指针是必须的。

具体代码如下：

typedef struct Node
{
    int data;
    struct Node *Next;
}Node, *List;
List L;
L = malloc (sizeof (Node));
if (L==null)
    printf ("Out of Memory!");
else 
    L->Next = null;

2.关于链表结构实现多项式的加法和乘法


用链表实现多项式的优点在于，省去了系数为0的项的存储空间和计算时间，操作方便。初步的想法是这样：

typedef struct Node
{
    int coef;
    int exp;
    struct Node *Next;
}Node, *Polynomial;

int AddPolynomial (Polynomial src, Polynomial dst);
int MultPolynomial (Polynomial poly1, Polynomial poly2, Polynomial polyMul);

加法的函数是将两个多项式的结果保存在一个中，当然这破坏了原有多项式的结构，不过为了计算乘法方便，所以采用这样的方式。需要注意的是多项式相加后系数为0时需释放空间。

乘法的函数则需要先初始化一个Temp链表和一个结果链表，然后每次循环将poly1的一个元素与poly2的每个元素相乘，由于Poly2中没有重复元素，所以无需判断，结果放到Temp链表中，每次循环结束时，调用相加的函数将Temp列表和结果链表相加，即可避免多项式的合并问题。为避免重复对内存进行分配和释放操作，可在头结点中记录当前链表的长度，提前分配Temp链表的空间，循环中只对值进行操作即可。

此想法未经代码验证。
